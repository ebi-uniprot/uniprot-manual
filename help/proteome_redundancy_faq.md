Methods to reduce proteome redundancy have been in effect since release 2015_04.
These methods were aimed at reducing the number of proteins in the UniProt Knowledgebase ([UniProtKB](https://www.uniprot.org/help/uniprotkb )) from redundant proteomes, in which all or nearly all protein sequences are highly similar or identical to an existing proteome from the same species. Reducing proteins from redundant proteomes prevented the unsustainable exponential growth of UniProtKB.

**As of release 2026_02 (May 2026)**, a comprehensive package of changes to our reference proteome pipeline (Link to’How are reference proteomes selected’ new help page) was finalized. The changes affect how [reference proteomes](https://www.uniprot.org/help/reference_proteome ) are selected and also what protein entries are included in UniProtKB. <br>
**The concept of proteome redundancy is no longer relevant and is therefore retired.**

For a summary of the changes to proteomes, please see our ‘[Quick summary of proteome changes](https://www.uniprot.org/help/refprot_only_changes )’ help page.
