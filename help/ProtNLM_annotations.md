UniProt’s [Automatic Annotation pipeline](https://www.uniprot.org/help/automatic_annotation) enhances [unreviewed/TreEMBL](https://www.uniprot.org/help/uniprotkb_sections) UniProtKB {https://www.uniprot.org/uniprotkb} entries with automatic classification and annotation.

UniProt leverages machine learning artificial intelligence (AI) using a Protein Natural Language Model (ProtNLM) developed by collaborators at [Google DeepMind] (https://deepmind.google/science/). ProtNLM was originally used to predict potential protein names for entries labeled as “uncharacterized protein” but has now been expanded to include predictions for function comments, subcellular locations, keywords, and Gene Ontology terms (ProtNLM2).

## ProtNLM methodology

ProtNLM uses an approach similar to AI natural language models that generate titles or captions for images. It is a [transformer-based model](https://storage.googleapis.com/brain-genomics-public/research/proteins/protnlm/uniprot_2022_04/protnlm_preprint_draft.pdf), trained on a snapshot of UniProtKB data to generate annotations for unreviewed/TrEMBL UniProtKB entries. Originally predictions were based solely on their amino acid sequence using a sequence-to-sequence model. Later models added the [organism TaxID](https://www.uniprot.org/help/taxonomic_identifier) and secondary structure predicted by [AlphaFold](https://alphafold.ebi.ac.uk/) to the inputs used to generate predictions.

Early iterations of ProtNLM data used a single model to generate predictions while later releases use an ensemble of multiple models, some taking different combinations of inputs, to improve performance.

## Timeline of ProtNLM generated data in UniProt

ProtNLM, trained on data from UniProt release 2021\_02 (including all RecNames, AltNames and SubNames), was used to predict protein names from amino acid sequence inputs for proteins with the name “Uncharacterized protein”. From UniProt release 2022\_04 onwards these predicted protein names have been included in reviewed/Swiss-Prot entry pages with the \[“Automatic Annotation (ECO:0008006) Google:ProtNLM” evidence tag\]([https://www.ebi.ac.uk/QuickGO/term/ECO:0008006](https://www.ebi.ac.uk/QuickGO/term/ECO:0008006)).  
ProtNLM has continued to develop over time:

* Release 2022\_04 \- ProtNLM used to predict protein names for proteins with the RecName “uncharacterized protein”.  
* Release 2022\_05 \- Predictions updated using an ensemble of 3 models that take only the amino acid sequence as input, and 3 models that take both the amino acid sequence and the organism TaxID as input.  
* Release 2023\_01 \- model score threshold introduced and improved post-processing to enhance prediction accuracy.  
* Release 2023\_02 \- A new ensemble of 7 models was trained on the original data including 1 model that uses AlphaFold secondary structure predictions as input.

![ProtNLM Ensemble Model](https://raw.githubusercontent.com/ebi-uniprot/uniprot-manual/refs/heads/main/images/protnlm2_1.png)

The second iteration of ProtNLM (ProtNLM2) is trained on 240 million proteins from UniProt release 2023_04, including both curator reviewed/Swiss-Prot entries and unreviewed/TrEMBL entries annotated by the UniProt Automatic Annotation pipeline. In addition to protein names, ProtNLM2 currently provides predictions for protein names, protein function comments, EC numbers, subcellular locations, keywords and Gene Ontology terms.

For protein name predictions ProtNLM2 has been trained on both recommended protein names as well as unreviewed submitted protein names, and this may be reflected in the predictions.

Annotations generated by ProtNLM2 for an initial 10K dataset have been made public in conjunction with the UniProt 2026_02 release. This includes predicted annotations for roughly 10,000 protein sequences from the 1,000 most searched species reference proteomes.

## ProtNLM prediction quality assessment

Predicted annotations have been continually assessed throughout the development of the ProtNLM models, both by automated evaluation as well as manual evaluation by expert UniProt biocurators working in close collaboration with Google DeepMind.

While ProtNLM models have been carefully evaluated during their development, machine learning AI models do make errors. We invite users to carefully evaluate these predictions and flag any issues or errors they find via the [UniProt help desk](https://www.uniprot.org/contact).

### ProtNLM model score

Each ProtNLM prediction is assigned a model score representing an estimate of the likelihood of the prediction being generated. This score reflects the model's probability of generating the prediction based on the input sequence. The score ranges from 0 to 1, with higher scores indicating a higher estimated likelihood.

For ProtNLM protein name ([RecName](https://www.uniprot.org/help/protein_names)) predictions we apply a model score threshold of 0.2, removing any predictions that fall below this.

For ProtNLM2 predictions we currently do not use a model score threshold and instead rely on a post-processing corroboration strategy to filter out unreliable predictions. Based on ongoing curator evaluation and user feedback on the 10,000 proteinK dataset release a model score threshold may be implemented for future ProtNLM2 releases. Model scores for all predictions are provided for the user to make their own assessment on prediction reliability.

### Post-processing corroboration strategy

As AI natural language models are in essence a black box system there is no way to determine what evidence the model uses to make a certain prediction. Instead we implement a post-processing corroboration system, designated the ‘Evidencer’, as a quality control pipeline for ProtNLM2 predictions. This process mimics the expert biocurator evaluation process.

Predicted annotations are corroborated by looking for exact or partial (substring) matches with other annotations in the entry being annotated. The Evidencer is also able to look for exact or substring matches in descriptions of cross referenced signatures within these entries from external databases such as InterPro (designated as a hydrated string/substring match). If no match is found the evidencer checks for sequence similarity (determined by the [phmmer tool](https://www.ebi.ac.uk/Tools/hmmer/search/phmmer) or structural similarity (determined by the [tm-align tool](https://www.rcsb.org/alignment)) to other entries in UniProtKB containing annotations identical to the predicted annotation. Predicted annotations are considered reliable if they have sequence similarity evidence with a phmmer bit score greater than 25 or structural similarity with a TM-align score greater than 0.5. Additionally structural similarity evidence requires an average pLDDT score of at least 70 across aligned residues.

We also apply an exclusion evidencer that removes annotations that meet exclusion criteria, such as GO annotations that are restricfted by [GO taxon constraints](https://geneontology.org/docs/taxon-constraints/#:~:text=Taxon%20constraints%20are%20used%20to,specificity%20of%20the%20GO%20terms) or inaccurate protein names such as those that do not conform to the [International Protein Nomenclature guidelines](https://www.uniprot.org/help/international_protein_nomenclature_guidelines).

The Evidencer workflow produces three possible outcomes:

1. **Approved and Displayed:** The ProtNLM2 annotation is corroborated by exact/substring or similarity Evidencers and not rejected by exclusion Evidencers. The annotation is approved and displayed in UniProt.  
2. **Rejected:** The ProtNLM2 annotation is corroborated but rejected based on matches to the unsuitable annotations list or Gene Ontology taxon constraints. The annotation is flagged and not displayed in UniProt.  
3. **Uncertain Quality:** The ProtNLM2 annotation is neither corroborated nor rejected, leaving prediction quality uncertain. The annotation is flagged and not displayed in UniProt.

## Accessing ProtNLM annotated data

## On the website

The ProtNLM predicted protein names for proteins previously named “uncharacterized protein” have been integrated into the UniProtKB data in a similar way as names predicted by other automatic annotation methods. These annotations can be recognised by the “Automatic Annotation (ECO:0008006) Google:ProtNLM” evidence tag, for example in entries  [A0A6E8VIU4](https://www.uniprot.org/uniprotkb/A0A6E8VIU4/entry#names_and_taxonomy) and [Q54TU2](https://www.uniprot.org/uniprotkb/Q54TU2/entry#names_and_taxonomy). To search or filter for all entries annotated with a ProtNLM predicted protein name use the search term “source:google” to filter for entries with [ProtNLM annotations](https://www.uniprot.org/uniprotkb?query=source%3Agoogle).

ProtNLM2 predicted annotations are not integrated into the UniProtKB entries directly but are stored in parallel and can be viewed on the website by switching on the AI annotation toggle found in the navigation pane of ProtNLM2 annotated entries. AI generated annotations are clearly differentiated by a purple background colour.  

### Interpreting prediction evidence information

We provide Evidencer information for ProtNLM2 predictions to help users evaluate their accuracy. These can be viewed by clicking the “Source: ProtNLM” tag next to the predicted information. This will open a text box containing evidence in support of the prediction. Several types of evidence statements are given:

* **Exact matches:** An exact string match of the prediction is found in the entry. Currently predictions that are an exact match to an annotation in an entry (eg a predicted function statement matching the function statement already in the entry) are not shown.
* **Substring match:** The predicted annotation partially matches a statement already in the entry.  
* **Hydrated substring match:** The predicted annotation partially matches the description of a term in an external database cross referenced in the entry.The cross-referenced databases queried using this approach include [InterPro/Pfam](https://www.ebi.ac.uk/interpro/), the [Gene Ontology (GO) knowledgebase](https://geneontology.org/), and Enzyme nomenclature database containing [EC (Enzyme Commission) numbers](https://enzyme.expasy.org/).  
* **Sequence similarity:** The predicted annotation is found in other entries with sequence homology to the entry. A link is provided to the entry with the highest similarity as determined by phmmer alignment. The higher the provided similarity bit score the more significant the sequence similarity.
* **Structure similarity:** The predicted annotation is found in other entries that share structural homology with the entry. A link is provided to the entry with the highest similarity as determined by TM-align. The TM-align score ranges from 0 to 1, with a score of 1 indicating a perfect match and structures with scores of \<0.2 considered unrelated while those with scores \>0.5 are considered to have a highly similar protein fold.

In the case of sequence and structure evidence, links are included to perform sequence alignments using the UniProt [Align tool](https://www.uniprot.org/align) or structural alignments using the FoldSeek [FoldMason tool](https://search.foldseek.com/foldmason) so  users can evaluate the prediction evidence themselves.
